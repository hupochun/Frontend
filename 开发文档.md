项目概述
一、问题背景与需求分析
在计算机技术快速迭代的背景下，程序员亟需高效的知识管理与共享机制。开源社区中，技术博客作为核心知识载体，不仅帮助开发者复用已有解决方案、避免重复劳动，还能沉淀碎片化经验、构建个人知识体系，并助力技术品牌建设。然而，当前许多个人博客平台功能单一、缺乏协同审核机制、内容组织混乱，难以满足团队协作、内容安全与高效检索等进阶需求。

为此，本项目拟开发一个面向程序员群体的技术博客管理系统，旨在提供一个结构清晰、权限分明、互动性强且具备内容审核能力的博客平台，解决以下核心问题：

用户身份与权限管理缺失：普通用户与管理员职责不清，缺乏安全的密码修改机制；
文章管理功能薄弱：缺少草稿保存、标签分类灵活管理、状态切换（公开/私密）等实用功能；
互动体验不足：评论、点赞、收藏等社交化功能未有效集成；
内容安全风险：用户生成内容（UGC）缺乏审核机制，存在违规信息传播隐患；
检索效率低下：无法按多维度（标题、标签、作者、时间、热度）高效检索技术文章。
二、相关研究与技术调研
通过查阅国内外关于博客系统、内容管理系统（CMS）、Web 应用架构及权限控制模型的相关文献，发现以下技术路线较为成熟：

前端框架：Vue.js / React 支持组件化开发，提升交互体验；
后端框架：Spring Boot（Java）或 Django（Python）提供完善的用户认证、RESTful API 支持；
数据库：MySQL 作为关系型数据库，适合存储结构化数据（用户、文章、评论、标签等）；
权限控制：基于角色的访问控制（RBAC）模型可清晰划分普通用户与管理员权限；
内容审核：结合关键词过滤、人工审核后台，实现半自动内容风控；
搜索优化：利用 MySQL 全文索引或集成 Elasticsearch 提升检索性能（本项目初期采用 MySQL 全文检索以简化架构）。
三、备选方案分析
方案	技术栈	优势	劣势	适用性
A. 单体架构 + Spring Boot + Vue + MySQL	Java, Vue, MySQL	开发成熟、文档丰富、团队易上手；权限与事务管理完善	扩展性有限，但满足课设规模	 高
B. 微服务架构 + Node.js + MongoDB	Node.js, Express, MongoDB	灵活、适合高并发	学习曲线陡峭，课设周期内难完成	低
C. 使用现成 CMS（如 WordPress）二次开发	PHP, WordPress 插件	快速上线	定制性差，不符合“自主开发”要求	不符合课设目标
经团队讨论，方案 A 在开发效率、技术可控性、功能完整性与课程要求契合度方面表现最优，故选定为实施方案。

四、最终方案确定
本项目将采用 B/S 架构，基于 Spring Boot（后端） + Vue.js（前端） + MySQL（数据库） 技术栈，实现一个具备以下核心能力的技术博客管理系统：

安全的用户登录与密码管理（含权限区分）；
完善的文章生命周期管理（草稿、发布、编辑、删除、公开/私密切换）；
灵活的分类与标签体系（用户可新建标签，管理员可维护分类）；
互动功能（评论、回复、点赞、收藏）；
内容审核机制（管理员审核文章、分类、标签）；
多维度检索与排序功能。
该系统不仅满足课程设计的功能与非功能要求，也为团队成员提供了实践全栈开发、数据库设计、前后端联调与软件工程协作流程的完整机会。

一、功能需求分析
（一）用户登录模块
用户认证功能：支持普通用户 / 管理员通过 “用户名 + 密码” 进行身份验证，验证失败时给出明确的 “用户名不存在” 或 “密码错误” 提示。
密码修改功能：用户（含管理员）可发起密码修改操作，需依次输入旧密码、两次新密码，系统校验规则如下：
旧密码与当前存储密码不一致时，提示 “旧密码输入错误”；
两次新密码输入不一致时，提示 “两次新密码不匹配”；
新密码需满足基础复杂度要求（可配置，如长度≥6 位、包含字母 + 数字），不满足时给出对应提示；
校验通过后，更新密码并提示 “修改成功”。
权限划分功能：系统基于角色划分权限，核心区分如下：
普通用户：仅操作个人账号、自有文章、评论互动等个人相关功能；
管理员：拥有普通用户全部功能，额外具备分类 / 标签管理、内容审核等系统级操作权限。
（二）文章管理模块
1. 普通用户文章操作功能
文章上传功能：支持编辑文章内容（富文本格式），可选择系统已有分类、关联标签（支持输入新建标签），提供 “保存草稿” 和 “提交发布” 两种操作，草稿仅本人可见，发布后进入待审核 / 公开状态（依系统配置）。
文章预览功能：文章发布 / 保存前，可预览最终展示效果（含排版、图片、格式等），预览页面与实际详情页一致。
自有文章管理功能：支持对个人发布 / 草稿文章进行批量删除、单篇编辑修改；支持文章状态切换（公开 / 私密），私密文章仅本人可见，公开文章需通过管理员审核后对外展示。
2. 管理员分类 & 标签管理功能
分类管理：支持新增文章分类（填写分类名称、描述）、编辑已有分类信息、删除无用分类（删除前校验是否关联文章，关联则提示无法删除）；
标签管理：支持新增标签、编辑标签名称、批量删除无效标签，支持查看标签关联文章数量。
（三）评论管理模块
评论发布与回复功能：用户在文章详情页可发布评论（支持文本格式），可针对他人评论进行层级回复，评论 / 回复提交后实时展示。
文章互动功能：用户可对文章执行点赞（单次账号仅可点赞一次，再次点击取消点赞）、收藏（收藏后加入个人收藏夹，取消收藏则移除）操作，点赞数、收藏数实时同步并展示在文章页面。
互动数据同步功能：所有点赞、收藏、评论操作数据实时写入数据库，页面无需刷新即可更新展示数据。
（四）内容审核模块
文章审核功能：管理员可查看待审核文章列表，支持单篇查看文章详情，执行 “通过审核”（文章转为公开状态）或 “驳回 / 删除” 操作（驳回需填写驳回原因，删除需确认操作，违规文章直接删除）。
分类 & 标签审核功能：用户新增的分类、标签需提交管理员审核，管理员可查看待审核分类 / 标签列表，执行通过或驳回操作，驳回需填写原因。
违规内容过滤功能：系统内置基础违规关键词库，审核文章、评论、分类 / 标签时，自动识别违规内容并高亮提示，管理员可针对性处理，支持手动添加违规关键词。
（五）检索模块
多条件检索功能：支持用户输入检索关键词，按 “文章标题”“关键词（文章内容）”“技术标签”“作者用户名” 四种维度进行精准检索，支持多维度组合检索。
检索结果排序功能：检索结果默认按 “发布时间降序” 排列，用户可手动切换排序维度（发布时间 / 阅读量）和排序方式（升序 / 降序），排序操作实时生效，无需重新检索。
检索效率优化功能：支持关键词模糊匹配，快速返回检索结果，展示文章标题、摘要、作者、发布时间、阅读量等核心信息，方便用户快速筛选。
二、非功能需求分析
（一）性能需求
响应时间：页面加载时间≤3 秒，登录、密码修改、评论 / 点赞等操作响应时间≤1 秒，文章检索（数据量≤1 万条）响应时间≤2 秒；
并发处理：支持同时在线用户数≥100 人，支持≥20 人同时执行检索、评论等操作，无明显卡顿；
数据存储：支持存储≥1 万篇文章、≥10 万条评论 / 回复数据，文章附件（图片等）存储容量≥10GB，数据读写效率稳定。
（二）可靠性需求
数据可靠性：用户账号信息、文章内容、互动数据等核心数据实时备份（每日自动备份一次，支持手动备份），防止数据丢失；
系统稳定性：系统连续运行时间≥72 小时无故障，出现异常（如数据库连接失败）时，给出友好提示并自动尝试恢复，不影响用户已有操作数据；
容错性：用户输入无效数据（如密码长度不足、格式错误）时，系统给出明确提示，不崩溃、不丢失已输入内容，支持表单数据暂存。
（三）易用性需求
界面友好性：采用简洁直观的技术博客风格界面，导航清晰，核心功能（发布文章、检索、评论）入口明显，符合程序员使用习惯；
操作便捷性：常用操作（批量删除文章、切换排序方式）步骤≤3 步，支持快捷键辅助操作（如 Ctrl+S 保存草稿），提供操作引导提示；
兼容性：B/S 架构支持主流浏览器（Chrome、Edge、Firefox），版本≥最新版前 3 个版本，页面自适应不同分辨率（电脑端、平板端）。
（四）安全性需求
账号安全：用户密码采用加密算法（如 MD5 + 盐值）存储，不明文保存；支持账号登录失败次数限制（连续 5 次失败锁定账号 30 分钟）；
权限安全：严格校验用户权限，普通用户无法访问管理员功能页面，无法修改 / 删除他人文章、评论；
内容安全：违规内容自动识别与过滤，防止不良信息传播；文章、评论发布前进行内容校验，防止恶意代码注入。
（五）可扩展性需求
功能扩展：预留功能接口（如文章附件上传、用户关注、文章分享），方便后续新增功能，无需大幅修改现有代码；
数据扩展：支持数据库表结构扩展，可新增字段（如文章阅读时长、用户积分），不影响现有数据查询与存储；
部署扩展：支持系统分布式部署，可根据用户量增加服务器节点，提升系统处理能力。
三、系统逻辑模型（E-R 图核心实体及关系）
（一）核心实体
用户（User）：属性包括用户 ID（主键）、用户名、密码（加密）、角色（普通用户 / 管理员）、注册时间、联系方式；
文章（Article）：属性包括文章 ID（主键）、标题、内容、分类 ID（外键）、作者 ID（外键）、发布时间、更新时间、状态（草稿 / 待审核 / 公开 / 私密）、阅读量、点赞数、收藏数；
分类（Category）：属性包括分类 ID（主键）、分类名称、描述、创建人 ID（外键）、创建时间、审核状态（待审核 / 已通过 / 已驳回）；
标签（Tag）：属性包括标签 ID（主键）、标签名称、创建人 ID（外键）、创建时间、审核状态（待审核 / 已通过 / 已驳回）；
文章 - 标签关联（Article_Tag）：属性包括关联 ID（主键）、文章 ID（外键）、标签 ID（外键）（用于实现文章与标签的多对多关系）；
评论（Comment）：属性包括评论 ID（主键）、文章 ID（外键）、评论人 ID（外键）、评论内容、评论时间、父评论 ID（用于回复功能，默认 0 为根评论）；
收藏（Collection）：属性包括收藏 ID（主键）、用户 ID（外键）、文章 ID（外键）、收藏时间（用于实现用户与文章的多对多收藏关系）；
点赞（Like）：属性包括点赞 ID（主键）、用户 ID（外键）、文章 ID（外键）、点赞时间（用于实现用户与文章的多对多点赞关系，保证单次点赞唯一性）。
（二）核心实体关系
用户（User）与文章（Article）：一对多关系（一个用户可发布多篇文章，一篇文章仅对应一个作者）；
用户（User）与分类（Category）：一对多关系（一个用户可创建多个分类，一个分类仅对应一个创建人）；
用户（User）与标签（Tag）：一对多关系（一个用户可创建多个标签，一个标签仅对应一个创建人）；
文章（Article）与分类（Category）：多对一关系（多篇文章可归属一个分类，一篇文章仅归属一个分类）；
文章（Article）与标签（Tag）：多对多关系（通过 Article_Tag 关联表实现，一篇文章可关联多个标签，一个标签可关联多篇文章）；
用户（User）与评论（Comment）：一对多关系（一个用户可发布多条评论，一条评论仅对应一个评论人）；
文章（Article）与评论（Comment）：一对多关系（一篇文章可有多条评论，一条评论仅对应一篇文章）；
用户（User）与文章（Article）：多对多关系（通过 Collection、Like 关联表实现，分别对应收藏、点赞关系）。
四、开发工具与技术选型
（一）整体架构选型
推荐采用 B/S（浏览器 / 服务器）架构，具备跨平台、易部署、维护成本低的优势，符合课程设计团队协作开发需求。
（二）前端技术栈
页面构建：HTML5 + CSS3（Tailwind CSS 简化样式开发）；
交互逻辑：JavaScript + Vue3（或 React，轻量易上手，适合团队快速开发）；
富文本编辑器：Wangeditor（开源免费，支持文章编辑、图片上传）；
前端工程化：Vite（构建工具，提升开发效率）；
UI 组件库：Element Plus（Vue3 配套组件库，提供丰富表单、列表、弹窗组件，减少重复开发）。
（三）后端技术栈
开发语言：Java（Spring Boot 框架）或 Python（Django/Flask 框架），推荐 Spring Boot（生态完善，易上手，适合团队协作）；
核心框架：
Spring Boot（快速搭建后端服务）；
Spring MVC（实现 MVC 分层，处理请求与响应）；
MyBatis-Plus（简化数据库 CRUD 操作，提升开发效率）；
辅助工具：
Hutool（Java 工具类库，简化加密、日期处理等操作）；
JWT（实现无状态用户身份认证）；
Redis（缓存热门文章、检索结果，提升系统响应速度）。
（四）数据库选型
主数据库：MySQL 8.0（开源免费，稳定性高，符合课程设计推荐要求，支持海量数据存储与高效查询）；
数据库管理工具：Navicat Premium（可视化操作，方便团队协作管理数据库、备份数据）。
（五）开发与部署工具
代码编辑器：VS Code 或 IntelliJ IDEA（支持代码高亮、团队协作、调试功能）；
版本控制工具：Git + Gitee/GitHub（实现团队代码协同开发，版本回溯）；
部署工具：Tomcat 9.0（Java 后端部署）或 Nginx（前端静态资源部署，反向代理后端服务）；
接口测试工具：Postman（测试后端接口可用性，方便前后端联调）。



课设内容和要求：
计算机技术知识密度高、更新快，程序员需持续积累经验才能立足。开源社区中，博客是知识流转核心载体——程序员可借其搜索相似问题方案，避免重复攻关。同时，博客能沉淀碎片化技术经验，助力构建知识体系、记录成长轨迹，还是分享经验、打造个人技术品牌的重要途径。
系统主要功能如下：
(1)	用户登录模块：用户能够通过用户名和密码登录系统，并能够对密码进行修改。在更改密码的时候，需要输入一次旧密码和两次新密码。如果密码输错会出现提示。同时，根据用户权限划分，分为普通用户与管理员。
(2)	文章管理模块：用户上传文章时，可选择系统已有分类、添加标签（支持新建标签），系统提供草稿保存功能，文章发布前支持预览效果；用户可对自有文章执行批量删除、状态切换（公开/私密）、编辑修改操作。管理员可对系统内的文章分类及标签进行新增、删除、修改操作。
(3)	评论管理模块：用户在文章详情页可对文章发布评论、回复他人评论；同时可对文章进行点赞、收藏操作，互动数据实时同步至系统。
(4)	内容审核模块：管理员可对用户发布过的文章进行审核，也可审核用户新增的分类、标签信息；审核过程中可识别并过滤违规内容，对违规内容执行删除操作。
(5)	检索模块：用户可使用系统检索功能，支持按文章标题、关键词、技术标签、作者信息检索内容。检索结果支持按发布时间、阅读量进行升序或降序排序，提升检索效率。
基本要求：
1、项目实施以团队为单位进行；
2、项目开发选择适当的面向对象的开发工具及数据库管理系统（推荐B/S程序或移动应用程序、MySQL数据库）；
3、团队共同制定开发计划，明确成员分工；
4、团队协同完成系统的需求分析、概要设计、详细设计、编码和测试；
5、团队按照课程设计规范书写并提交一份课程设计报告；
6、成员各自提交一份课程设计总结。
